\begin{abstract}
本报告围绕"全链路3D视觉挑战"展开，涵盖真实多视角重建、文本驱动3D生成、
单图到3D生成三条技术路径，并将所得资产融合至重建的真实背景场景中；同时还
包括具身智能领域基于ACT算法的动作策略跨环境泛化实验。
在3D资产准备阶段：物体A采用手机拍摄视频，经由COLMAP提取120帧的精确位
姿（重投影误差0.54 px），再以3D Gaussian Splatting（gsplat 1.5.3）训练50\,000
轮，最终获得594\,197个高斯球，训练用时仅\textbf{13.5分钟}（NVIDIA H20），
全视角均值PSNR为\textbf{33.00 dB}；物体C采用单张真实照片（九尾狐手办），经
背景去除后输入Zero123++（19.7 s，生成6个新视角），进而由TripoSR在12.0秒内重建
带纹理网格；物体B由队友利用threestudio框架的SDS Loss从文本Prompt生成（见
第\ref{sec:objB}节）。
质量对比实验表明，多视角重建（PSNR 33 dB）在几何精度和纹理细节上最优，
但需要充分的输入视角覆盖和较长训练时间；文本生成灵活性最高但存在Janus问题，
可通过Perpendicular Negative引导技术有效缓解；单图生成仅需单张照片且推理时间
仅32秒，在便利性与质量之间取得了较好平衡。
\end{abstract}

\section{背景介绍}
\label{sec:intro}
%% ============================================================

三维场景理解与资产生成是计算机视觉和图形学的核心问题，近年来随着神经辐射
场（NeRF）\cite{mildenhall2020nerf} 和3D高斯溅射（3DGS）\cite{kerbl20233d} 的
快速发展，从图像到高质量3D表达的技术路径日趋多样。

本次作业聚焦于以下三条互补的3D资产生成路径：
\begin{enumerate}
  \item \textbf{真实多视角重建（物体A）}：借助COLMAP \cite{schoenberger2016sfm}
        从视频中恢复稠密点云和相机位姿，再以3DGS将其细化为可微分的高斯表达，
        从而支持实时高质量渲染。
  \item \textbf{文本到3D生成（物体B）}：以threestudio \cite{threestudio2023} 为
        框架，利用Score Distillation Sampling（SDS）\cite{poole2022dreamfusion}
        将预训练2D扩散模型的知识蒸馏至3D表达中，无需任何输入图像即可从文本
        描述生成3D虚拟资产。
  \item \textbf{单图到3D生成（物体C）}：先用rembg去除背景获得纯净前景，再
        经Zero123++ \cite{liu2023zero} 生成多视角假设图像，最后用TripoSR
        \cite{tochilkin2024triposr}重建完整3D模型。
\end{enumerate}

同时，我们将上述资产融合至基于Mip-NeRF 360数据集 \cite{barron2022mipnerf360}
重建的背景场景，并在题目二中探索基于ACT \cite{zhao2023learning} 的动作策略
在跨环境泛化中的表现。

\subsubsection{3D Gaussian Splatting训练}

\paragraph{高斯初始化}
以COLMAP稀疏点云作为高斯球的初始位置，共$N_0 = 34\,192$个高斯球。
\begin{itemize}
  \item \textbf{位置}（means）：直接取点云坐标。
  \item \textbf{颜色（SH系数）}：由点云RGB颜色反算DC分量（$\text{SH}_{dc} =
        (c - 0.5) / C_0$，$C_0=0.28209$），高阶项初始化为0。使用 3 阶球谐函数
        ($L=3$，共16个基函数，每通道)。
  \item \textbf{缩放}（scales）：初始化为 $\log(\bar{d}) = \log(0.1051)$，各向同性。
  \item \textbf{不透明度}（opacities）：$\text{logit}(0.1)$，避免初始过不透明。
  \item \textbf{旋转}（quats）：恒等四元数 $[1, 0, 0, 0]$。
\end{itemize}

\paragraph{损失函数}
结合L1损失与SSIM损失：
\begin{equation}
  \mathcal{L} = (1-\lambda_{\text{SSIM}}) \cdot \mathcal{L}_1 +
                \lambda_{\text{SSIM}} \cdot \mathcal{L}_{\text{DSSIM}},
  \quad \lambda_{\text{SSIM}} = 0.2
\end{equation}

\paragraph{自适应密度控制（ADC）}
采用gsplat的 \texttt{DefaultStrategy}，关键设置如表\ref{tab:adc}所示。
场景尺度 $s_{\text{scene}} = \max(\text{bbox\_extent})/2 = 22.75$ m。
分裂/克隆阈值为 $\tau_{\text{scale}} = 0.10 \times s_{\text{scene}} = 2.275$ m；
梯度阈值 $\tau_{\text{grad}} = 8\times10^{-4}$（使用绝对梯度 absgrad=True）。

\subsubsection{实现细节与调试经验}

在实际训练中遇到的主要问题及解决方案如下：

\begin{enumerate}
  \item \textbf{SH格式问题}：gsplat的 \texttt{rasterization()} 要求颜色输入为
        $[N, K, 3]$ 格式（$K$为SH系数数），而非展平的 $[N, K\cdot3]$。错误
        格式会引发 \texttt{AssertionError}。
  \item \textbf{梯度保持问题}：DefaultStrategy的 \texttt{step\_post\_backward()}
        需要访问 \texttt{info["means2d"].grad}，但该张量是非叶子节点，默认不
        保留梯度。正确做法是在 \texttt{loss.backward()} 前调用
        \texttt{strategy.step\_pre\_backward()}，由策略自动调用
        \texttt{retain\_grad()}。
  \item \textbf{高斯球爆炸问题}：设置 \texttt{grow\_scale3d=0.01}（默认值）
        时，分裂阈值 $0.01 \times 1.0 = 0.01$ 远小于初始高斯球尺度
        $\bar{d}\approx0.10$，导致所有高斯球每步均被分裂，迭代12K时数量从
        34K激增至18.7M，PSNR崩溃至3 dB。修复方案：将
        \texttt{grow\_scale3d} 调整至0.10，并将
        \texttt{scene\_scale}设为点云包围盒对角线的一半（22.75 m）。
  \item \textbf{absgrad一致性}：当 \texttt{rasterization(absgrad=True)} 时，
        DefaultStrategy应同样设置 \texttt{absgrad=True}，并将
        \texttt{grow\_grad2d} 相应提高至 $8\times10^{-4}$（见gsplat文档建议）。
\end{enumerate}

\subsection{物体B：文本到3D生成（threestudio + SDS + Perp-Neg）}
\label{sec:objB}

\subsubsection{方法：Score Distillation Sampling}

threestudio \cite{threestudio2023}基于Score Distillation Sampling（SDS）损失
\cite{poole2022dreamfusion}，利用预训练2D扩散模型（Stable Diffusion 1.5）对3D
体积场（隐式NeRF + DMTet几何）的梯度信号进行反向传播，将3D表达向文本描述对齐。
无需任何真实图像即可生成三维内容。

\subsubsection{Janus问题与Perpendicular Negative修复}

SDS训练的常见缺陷是\textbf{Janus problem}（多面怪）：扩散模型对不同摄像机方向
缺乏位姿感知，导致物体背面/侧面也渲染成正面的外貌。v4版本（guidance scale=60）
仍出现柯基犬背面可见面孔的现象。

v5引入两项关键修复：
\begin{enumerate}
  \item \textbf{View-Dependent Prompting}：threestudio根据当前渲染摄像机方位角/仰角，
        自动在prompt前附加 ``front view of''、``side view of''、``back view of'' 或
        ``overhead view of''，使扩散模型感知视角方向。
  \item \textbf{Perpendicular Negative（Perp-Neg）}：对侧面摄像机施加垂直方向的
        负引导梯度，专门抑制侧视角出现正面特征，是目前最有效的Janus问题缓解方案之一
        \cite{armandpour2023re}。
\end{enumerate}

\subsubsection{Zero123++：扩散先验驱动的新视角合成}

Zero123++ \cite{liu2023zero}在Zero123的基础上引入多视角联合预测架构，以单张
RGBA图像为条件，一次性生成固定方位角/仰角网格下的6个新视角。关键改进包括：
（1）多视角注意力机制保证跨视角一致性；（2）全局-局部相对位姿编码；
（3）通过参考注意力将输入图语义注入所有生成视角。

本实验使用官方 \texttt{sudo-ai/zero123plus-v1.2}，通过HuggingFace diffusers
流水线加载，推理步数75步，在H20 GPU上实测仅需 \textbf{19.7秒}即可生成6个新视角
（模型权重3.46 GB，首次下载后缓存，后续无需重下）。

\subsubsection{TripoSR：前馈式单图3D重建}

TripoSR \cite{tochilkin2024triposr}基于大型Transformer网络，以单张图像为输入，
直接回归三平面（triplane）NeRF表达，再通过Marching Cubes（分辨率256）提取
带纹理的三角网格。该方法无需迭代优化，推理仅需约10秒。

本实验以去背后的 \texttt{object\_C\_256\_rgba.png} 为输入，设置前景占比0.85，
最终输出带纹理的 \texttt{mesh.obj}（8.5 MB，约3万面片）。由于输入已去背景，
推理时添加 \texttt{--no-remove-bg} 跳过TripoSR内置的去背景步骤。

\subsection{现象分析}

\paragraph{多视角重建的优势与局限}
3DGS通过对每个高斯球的颜色、位置、形状进行像素级监督，能够以极高保真度
重现物体表面细节——前提是输入视角充分覆盖目标区域。本实验中物体A底部区域
覆盖较少，可能导致该区域的高斯球初始化不充分，渲染时出现漏洞（floater）
或颜色噪声。此外，3DGS依赖于精确的相机位姿；COLMAP对视觉纹理丰富的场景
效果好，对纹理均匀物体（如白色塑料）匹配较差。

\paragraph{文本生成的优势与局限}
SDS Loss的本质是将2D扩散模型的分数函数作为3D表达的监督信号，天然继承了
扩散模型对语义概念的理解，无需任何真实图像输入，适合生成虚拟/不存在的物体。
然而，SDS Loss存在两大典型缺陷：（1）"过度平滑"（over-smoothing），高频纹理
细节不足；（2）"Janus问题"（Janus problem），即从正面视角渲染时反面也会错误地
呈现为正面，导致物体多个方向均有"脸"。本实验中，v4版本的柯基犬出现了三只耳朵
及背面出现面孔的现象。为此，v5引入\textbf{Perpendicular Negative（Perp-Neg）}
梯度技术，结合view-dependent prompting（自动为每个摄像机方向附加"front/side/back
view of"前缀），显著缓解了Janus问题，最终生成了解剖学上更合理的柯基犬3D模型。

\paragraph{单图生成的优势与局限}
Zero123++以单张图像为条件生成多视角，大幅降低了3D内容生产门槛。其核心限制
在于：生成的新视角仍是通过扩散模型"想象"出来的，对于遮挡区域（如物体背面）
可能出现幻觉（hallucination），导致重建的网格在视角外区域细节不一致。此外，
TripoSR是前馈式推理，无迭代优化，对输入图像的角度和背景质量较敏感。

\paragraph{方法概述（占位）}
ACT（Action Chunking with Transformers）\cite{zhao2023learning}将机器人动作预测
建模为序列生成问题，通过Transformer架构实现对一段未来动作序列（chunk）的联合

与模仿学习相比，动作分块机制的优势在于：即使单步预测存在轻微偏差，在同一
chunk内的平滑约束可以抑制高频噪声，提升跨环境视觉偏移下的鲁棒性。

\section{结论}
\label{sec:conclusion}
%% ============================================================

本工作系统对比了三种3D资产生成技术路径，验证了各自的优势与局限：
多视角重建质量最高但依赖多角度采集；文本生成灵活但几何精度受限；
单图生成取得了便利性与质量的良好平衡。场景融合实验表明，将不同表达形式
（高斯球 + 网格）统一至同一渲染框架的关键挑战在于坐标系对齐和几何质量的
一致性。未来工作可以探索基于GaussianEditor或Gaussian Grouping等方法，实现
更无缝的多物体高斯场景融合。
